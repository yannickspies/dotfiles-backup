MyZone - Create dashboard page - Your insurances

```
Story Goal

We want to have the first page of the MyZone (Dashboard) where customers can see their insurances and can quickly navigate to the actions they want to do. The Dashboard page is created with the new design components.

Required action / Scope
Create the first section "Your insurances" in the Dashboard in the new Angular app (multilingual NL/DE/ES).
Use the new design components from the Storybook component library
Connect the components with the Angular functionality that's already in place for the current MyZone
Content sections:

Create section: "Your insurances"

Title is a H2
Get the data from WS:
/en-gb/relation-general/request/relation_data
/en-gb/relation-general/request/policy
Show data from WS at the right places in component created in WINS-13604
Show policynumber from relation_data
Show ('ACTI', 'TOEK', 'ACEI', 'NEVT', 'OSTP', 'ACWI') insurances as designed
All possibilities from validity_status_code:
ACTI (actief): show products with green label "Actief"
TOEK (toekomstig): show products with orange label "Toekomstig"
INAC (inactief) don't show
EIND (beëindigd): don't show
ACTO (actief of toekomstig) don't show
ACEI (actief en beëindigd) show products with orange label "?"
ACWI (actief met wijziging) show products with green label "Actief"
NEVT (toekomstig product met een einddatum) show products with orange label "?"

(Details coverage in: en-gb/insurance-coverage/calculate/coverage-premium-purchased_product. Used on detail page)

Acceptance Criteria
The "Your insurances" section in the dashboard is styled like the design (https://www.figma.com/design/7wU69VagekV41zKmMfCvDD/My-InShared-Redesign--Marian-s-version-?node-id=2097-10089&t=jkOb6BGRXmMDWbrF-0)
Show the name of the customer at the top
The section "Your insurances" is created and filled with the content from the backend
All the insurances of the customer are shown (with status ACTI, TOEK)
The navigation will be implemented in a later state (WINS-13581)
The chatbot will be implemented in a later state (no ticket yet)
The message sections (https://www.figma.com/design/7wU69VagekV41zKmMfCvDD/My-InShared-Redesign--Marian-s-version-?node-id=2097-10064&t=jkOb6BGRXmMDWbrF-0) will be implemented in a later state (no ticket yet)
The implementation works on mobile and desktop (IE, FF, Chrome, iPhone, Samsung etc.)
Validation / Proof

How do we show that it works?

Unit Tests
Component renders the title "Your insurances"
Component loads backend data via the existing Angular services
Mapping of validity_status_code → label color and text is tested
Display of the policy number is tested
E2E Test

Tests the full flow:

Login → Dashboard
Loading data via relation_data and policy endpoints
Verifying that all insurances with status 'ACTI', 'TOEK', 'ACEI', 'NEVT', 'OSTP', 'ACWI' are displayed
Verifying that the labels and styling match the design
Verifying that other functionality (navigation/chatbot) is not present
 ```
